# Telegram-бот для выбора TradingView символа с самой длинной историей

Бот принимает тикер криптовалюты и выбирает биржу с максимально ранней историей цены, чтобы сразу открыть лучший символ в TradingView.

## Что делает бот

- находит монету по тикеру через CoinGecko;
- собирает торговые пары с CoinGecko, CryptoCompare и биржевых API;
- определяет самую раннюю доступную историю по бирже и паре;
- при одинаковой дате старта сравнивает качество часового графика: разрывы и плоские свечи;
- при прочих равных учитывает объём торгов;
- кэширует готовый результат на 24 часа.

## Важное замечание

Для полной логики с максимально широкой исторической проверкой нужен `CRYPTOCOMPARE_API_KEY`. Без него бот всё равно работает, но самые старые истории на биржах с ограниченными OHLC API могут определяться не полностью.

## Структура проекта

- `bot/` — точка входа и Telegram-хендлеры
- `config/` — загрузка настроек
- `services/` — API-клиенты, кэш и оркестратор поиска
- `utils/` — модели, форматирование, нормализация
- `data/` — файл JSON-кэша во время работы

## Установка

1. Создайте и активируйте виртуальное окружение:

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
```

2. Установите зависимости:

```powershell
pip install -r requirements.txt
```

3. Создайте `.env` на основе примера:

```powershell
Copy-Item .env.example .env
```

4. Заполните минимум:

- `BOT_TOKEN`
- `CRYPTOCOMPARE_API_KEY`

`COINGECKO_API_KEY` можно оставить пустым, но он полезен при жёстких rate limit.

## Запуск

```powershell
python -m bot.main
```

## GitHub и Railway

Проект уже подготовлен для деплоя через GitHub в Railway.

- `Procfile` запускает бота как `worker`
- `railway.json` содержит команду старта
- `BOT_TOKEN` должен быть задан в переменных Railway
- подробная инструкция лежит в `deploy/github-railway/README.md`
